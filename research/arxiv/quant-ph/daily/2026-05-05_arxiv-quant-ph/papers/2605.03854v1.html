Space-Time Tradeoffs of Pauli-Based Computation in Distributed qLDPC Architectures

原摘要

Pauli-based computation (PBC) provides a universal framework for executing fault-tolerant quantum algorithms using Pauli测量 and magic states. In monolithic architectures, the serialized nature of PBC directly ties runtime to a circuit's T-gate count, making it slow on metrics like circuit depth. However, in distributed quantum computing (DQC), the primary bottleneck is remote Bell pair generation. We investigate the tradeoff between error-correcting code block size and execution time of PBC within the Q-Fly architecture at intermediate scale, limiting individual node capacities to reflect near-term constraints while supplying abundant network nodes to minimize routing and compilation effects. We find that large qLDPC code blocks outperform the surface code baseline in terms of execution time by up to an order of magnitude when evaluated against quantum优化 algorithms. By moving groups of qubits to free nodes to bypass the sequential bottleneck of PBC, the large-block architecture minimizes网络 operations and achieves faster overall execution. This demonstrates that PBC is a competitive model in the distributed regime, establishing it as a practical compilation baseline for qLDPC systems before invoking more efficient transversal or homological gates.

分布式量子计算与量子纠错编译方向。该工作研究分布式架构中Pauli-based计算（PBC）的时空权衡，发现在Q-Fly架构中使用大码块qLDPC编码可将执行时间比表面码基线提升一个数量级。核心策略是通过将量子比特组迁移到空闲节点来绕过PBC的顺序瓶颈，最小化网络操作。对于关注量子编译控制和分布式量子计算的研究者，该工作为qLDPC系统的实用编译基线提供了分析，揭示了分布式环境下PBC的竞争力，对量子纠错码选择和编译策略优化有参考价值。

核心物理图像

在分布式量子计算中，多个量子计算节点通过量子网络连接，每个节点运行纠错码保护逻辑量子比特。Pauli-based计算通过顺序的Pauli测量和magic态注入实现通用计算，在单体架构中这导致运行时间与T门数量成正比。该工作发现，在分布式架构中可以通过将量子比特组动态迁移到空闲节点来绕过顺序瓶颈，同时使用大码块qLDPC编码减少所需的网络操作次数，从而将整体执行时间比表面码基线提升一个数量级。
